## 2026-04-06 - [Lazy Indexing for Static Data Traversal]
**Learning:** Frequent $O(N)$ traversals of the hierarchical `SUBJECTS` object in `data.js` were a significant bottleneck. Functions like `getFormulaById` were re-traversing the entire tree on every call, even though the data is static. The relation resolver also uses a 5-character prefix fallback which is $O(N)$ and can return surprising matches (e.g., 'Electric Field' matching 'Electromagnetic Spectrum').
**Action:** Implement lazy-initialized Map-based indexes during the first call to `getAllFormulas`. Store subject-specific and global names separately in the index to preserve the original lookup priority (subject-match > global-match) in $O(1)$. Prefix-based fallbacks should remain $O(N)$ or be carefully indexed if they become a performance issue later.
